# DEFINITION_OF_DONE.md

## Purpose

Defines completion criteria for each page type. A page is not done until it meets its criteria and passes the six quality gates in `REVIEW_CHECKLIST.md`.

## Status

Planned

## Priority

P1

## Owner

Shared

---

## Universal Criteria (all pages)

- [ ] Answers one executive hiring question
- [ ] Teaches, proves, clarifies, or demonstrates judgment
- [ ] Passes the six quality gates
- [ ] Follows `DESIGN.md`
- [ ] Responsive; no horizontal overflow
- [ ] Accessible (semantic HTML, focus states, contrast, reduced-motion)
- [ ] No portfolio/case-study language (see `VOCABULARY.md`)
- [ ] Connects to another page (no dead end)
- [ ] No unsupported claims

---

## Homepage

- [ ] Passes the 30-second recruiter test
- [ ] Lands identity in 10 seconds (Layer 1)
- [ ] Surfaces proof without deep reading (Layer 2)
- [ ] Clear path into Capabilities and Transformations

## Capability Page

- [ ] Frames a capability, not a project
- [ ] Backed by evidence and decisions
- [ ] Links to relevant transformation stories and frameworks

## Transformation Story

- [ ] Follows the 12-part structure in `TRANSFORMATION_STORIES/FRAMEWORK.md`
- [ ] Communicates transformation, not process
- [ ] Metrics pass Claims Review
- [ ] States Nikhil's specific contribution

## Essay

- [ ] Teaches a reusable idea or framework
- [ ] Survives the "remove the name" test
- [ ] Links to related thinking and frameworks

## Operating Manual Page

- [ ] Documents a system, not an opinion
- [ ] Reusable by another leader
- [ ] Offers a downloadable artifact where relevant

## Builder Page

- [ ] Shows product judgment and outcomes
- [ ] Distinguishes what Nikhil built personally
- [ ] Links to relevant transformation stories

## Contact Page

- [ ] Clear, low-friction path to a conversation
- [ ] Reinforces executive framing
- [ ] Sets expectations for who should reach out

## Navigation

- [ ] Clear at all breakpoints
- [ ] Reflects the core nav order
- [ ] No dead links

## Footer

- [ ] Supports wayfinding across the knowledge system
- [ ] Human footer note only after executive frame is established
