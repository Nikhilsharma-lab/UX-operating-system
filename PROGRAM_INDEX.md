## 1. Synthesis

This is not a UX portfolio. It is an executive knowledge system, operating manual, transformation archive, writing system, and AI-native product leadership platform.

The first visible product of this knowledge system is a website. The website is the first vertical slice of a much larger, long-running career platform.

The entire system revolves around one thesis:

**Building AI-native product organizations that transform ambiguity into measurable business outcomes.**

The system is organized around two views:

- **The Map** — the complete, exhaustive program index. Nothing is forgotten.
- **The Route** — the ruthless release sequence. What ships first, and why.

The Route over the Map is a core project decision. The Map guarantees nothing is lost. The Route guarantees something ships.

## 2. Operating Model

This project uses multiple AI tools, but they do not have equal roles.

### Core Rule

ChatGPT owns the thinking.

Claude Code owns the implementation.

Nikhil owns the final decision.

| Function | Seat | Owns |
|---|---|---|
| **Founder / Final Decision Maker** | Nikhil Sharma | Truth, taste, ambition, approvals, raw career context, final judgment |
| **Creative Director / Strategy / Prompt Architect** | ChatGPT | Positioning, strategy, IA, writing, storytelling, page briefs, Claude Code prompts, critique, quality review |
| **Implementation Engineer** | Claude Code | Code, components, pages, motion, accessibility, performance, SEO wiring, refactors, bug fixes |
| **Technical Second Opinion** | Claude Chat, optional | Debugging, code explanation, isolated technical architecture reviews only |

### The Core Loop

1. Select a work item from `PROGRAM_INDEX.md`.
2. ChatGPT creates the strategy, copy direction, quality criteria, and Claude Code implementation prompt.
3. Claude Code implements only the assigned scope.
4. Nikhil brings screenshots, diffs, or output back to ChatGPT.
5. ChatGPT reviews against the six quality gates.
6. Nikhil approves or rejects.
7. Update `DECISION_LOG.md`.
8. Move to the next item.

### Decision Rule

When ChatGPT and Claude Code disagree:

- ChatGPT wins on strategy, narrative, brand, copy, IA, and creative direction.
- Claude Code wins on implementation feasibility, code quality, performance, and maintainability.
- Nikhil makes the final decision.

## 3. Operating Principles

1. **This is not a portfolio.** It is an executive knowledge system.
2. **Capabilities over projects.** Projects expire. Capabilities compound.
3. **Transformation stories over case studies.** Show transformation, not process.
4. **Every page answers one executive hiring question.** If it does not, it should not exist.
5. **Teach before impressing.** The visitor should leave smarter.
6. **Evidence over adjectives.** Use metrics, outcomes, decisions, and tradeoffs.
7. **Evidence before exposure.** No major metric, award, business claim, or internal product claim goes public until it is captured in `EVIDENCE/CLAIMS_REGISTER.md` and passes Claims Review.
8. **The project remembers.** Any important idea from strategy conversations must be captured in either `PROGRAM_INDEX.md`, `MASTER_PLAYBOOK.md`, `DECISION_LOG.md`, `BACKLOG.md`, or the relevant folder file before implementation proceeds.

## 6. Quality Systems

Six quality gates, each a single question (see `QUALITY/`):

1. **Executive Review** — Does this reinforce Nikhil as an AI-native product executive?
2. **Story Review** — Does it communicate a transformation, not just a project?
3. **Craft Review** — Is the writing, layout, spacing, typography, and visual rhythm excellent?
4. **Evidence Review** — Are claims backed by outcomes, metrics, decisions, or proof?
5. **Evergreen Review** — Will this still feel useful and credible five years from now?
6. **Compounding Review** — Can this idea become an essay, talk, post, video, framework, or course module?

Plus audience-specific gates: Claims Review, Confidentiality Review, Hiring Manager Review, Recruiter Review, Founder Review.
